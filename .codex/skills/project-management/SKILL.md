# Project Management

Keep the project small and directional.

## Rules

- Update `docs/PROJECT.md` when the goal or next milestone changes.
- Prefer one active milestone at a time.
- Write decisions as short notes, not long architecture documents.
- Split work into visible increments that can be tested.
- Avoid adding infrastructure before it supports an active milestone.

## Current Shape

This project begins as a minimal FastAPI backend. Add complexity only when a user-facing feature asks for it.
